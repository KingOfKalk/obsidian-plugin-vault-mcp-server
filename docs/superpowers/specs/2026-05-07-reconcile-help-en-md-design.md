# Reconcile `docs/help/en.md` with the audited PRD

**Issue:** [#318](https://github.com/KingOfKalk/obsidian-plugin-vault-mcp-server/issues/318)
**Date:** 2026-05-07
**Refs:** #264 (PRD audit that surfaced the divergences)

## Problem

The user manual at [`docs/help/en.md`](../../help/en.md) has drifted from the
audited PRD on five user-facing surfaces. Two are user-misleading (stale auth
framing, dead UI tools listed). The remaining three are stale tool counts,
new tools not surfaced in the catalogue, and inconsistent repo URLs in the
install / bug-report flows.

## Goal

Make every user-facing claim in `docs/help/en.md` match what the plugin
actually ships, with the smallest possible diff. No behaviour changes; no
re-structuring of the doc.

## Non-goals

- Re-auditing the PRD itself — already done in #264.
- Translating to other locales — only `en.md` exists today.
- A broader copy-edit pass — out of scope per the issue's audit boundary.
- Restructuring the Feature Modules table or adding a "What's new" callout —
  the user chose "extend module descriptions only".
- Updating `CHANGELOG.md` — auto-generated by release-please from frozen
  historical releases; the typo'd URLs there are not in scope.

## Edits

### Edit 1 — Auth elevator pitch (line 26)

The body already describes auth-on-by-default with auto-generated keys and
the `iAcceptInsecureMode` opt-out (lines 97, 152–153, 419–424). Only the
one-line summary at the top is wrong.

**Before:**
```
- **Auth**: opt-in HTTP Bearer token
```

**After:**
```
- **Auth**: HTTP Bearer token, on by default
```

### Edit 2 — Feature Modules table (lines 204–210)

Update tool counts and descriptions to match
[`docs/tools.generated.md`](../../tools.generated.md). Editor (10),
Workspace (5), and Templates (3) rows are already correct and stay
unchanged.

| Row                       | Count: before → after | Description change                                                                                                                                                            |
| ------------------------- | --------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Vault and File Operations | 16 → **18**           | append `, daily note` to the trailing fragment (`…binary I/O, metadata, daily note.`)                                                                                         |
| Search and Metadata       | 12 → **6**            | rewrite to drop the aspect-getter capabilities (now in `vault_get_aspect`): `Full-text search, list tags, find notes by tag / frontmatter, find resolved / unresolved links.` |
| UI Interactions           | 3 → **1**             | `Show notices and confirm/prompt modals.` → `Show notices.`                                                                                                                   |
| Plugin Interop            | 5 → **6**             | `List plugins, run Dataview / Templater queries, execute commands.` → `List plugins, run / describe Dataview queries, describe Templater templates, execute commands.`        |

The new tools (`vault_daily_note`, `plugin_templater_describe_template`,
`plugin_dataview_describe_js_query`) are surfaced *only* through the module
description text — readers wanting the full per-tool list follow the
existing pointer to `docs/tools.generated.md` (already cross-linked from
the codebase).

### Edit 3 — Repo URL consistency (lines 62, 488)

Canonical URL is `https://github.com/KingOfKalk/obsidian-plugin-vault-mcp-server`
(confirmed via `gh repo view`; matches `README.md` and lines 79 / 287 of
`en.md`).

**Replace** `https://github.com/KingOfKalk/obisdian-plugin-mcp` with the
canonical URL at:

- Line 62 — BRAT install instructions.
- Line 488 — bug-report URL in the FAQ.

Lines 79 (release link) and 287 (source-of-truth link) already use the
canonical URL — no change.
