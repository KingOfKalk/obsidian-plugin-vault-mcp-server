# `vault-mcp-server` Rename Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Complete the rename deferred from #247 — change the plugin id, display name, npm package name, GitHub repo, release zip name, log prefix, and all docs/test references to `vault-mcp-server`, while keeping the AI-client-facing `serverInfo.name` (`obsidian-mcp-server`) intentionally distinct.

**Architecture:** Mechanical rename, no behaviour change. Three logical commits on a single branch: (1) plugin id + package + tests + log prefix; (2) CI workflows; (3) docs. Repo rename happens out-of-band on GitHub before branching. No migration shim (the maintainer is the only installer; manual folder rename once).

**Tech Stack:** TypeScript 6.x, Vitest 4.x, ESLint 10.x, Obsidian Plugin API, GitHub Actions, release-please.

**Spec:** [`docs/superpowers/specs/2026-05-04-vault-mcp-server-rename-design.md`](../specs/2026-05-04-vault-mcp-server-rename-design.md)
